Required Additional Method: split_data

  • Functionality:
    This method splits the dataset into a training and a test set. The training set will later be used to train a classifier; the test set will be used to evaluate the predictive performance of a classifier. You can use the method train_test_split [3] from sklearn.model_selection in your implementation of split_data.

    The method split_data uses the following information provided in the config file:

    • random_state_for_split: The random state for the train/test split. Setting a random state allows for reproducible output (see [3] for further details).

You can use sklearn.impute.SimpleImputer [4] to do this. The following imputation strategies (i.e., allowed values for the parameter impute_strategy) should be supported:

    • "mean": This strategy replaces any missing values of a feature by the mean of that feature in the training set (also see [4]).
    • "median": This strategy replaces any missing values of a feature by the median of that feature in the training set (also see [4]).
    • "most_frequent": This strategy replaces any missing values of a feature by the most frequently occurring value of that feature in the training set (also see [4]).
    • None: Completely removes all rows where any feature has a missing value.

Note: You will learn about this algorithm in detail in the tutorials. There is also a basic explanation here and a more thorough and mathematical one on Wikipedia.

Class SimpleBaselineClassifier (4P)

The SimpleBaselineClassifier makes trivial predictions that do not use any information from the features in the training data. Such a baseline is useful for determining whether (and by how much) the predictive performance of other, more complex classifiers is better than what could be achieved with the most naive approach.
Note: The SimpleBaselineClassifier implements some of the functionality of sklearn’s DummyClassifier. You can use the DummyClassifier as a reference for more information on the strategies.

You should implement the following strategies:

  • most_frequent: This strategy always predicts the most frequent value of the target variable in the training data.
  • uniform: This strategy predicts the value of the target variable at random, chosen from the set of unique values that occur in the (target variable of the) training data.
  • constant: This strategy always predicts the same value.

Required Method: __init__

  • Functionality:
    This method stores the value of the parameters in a public instance attributes of identical names.

  • Parameters:

    Name Type Default Value Description
    strategy str 'most_frequent' The strategy that the SimpleBaselineClassifier should use to make simple predictions. Possible values: most_frequent, uniform, constant.
    random_state int None Used for reproducible output when using the strategy uniform.
    constant int or str None If the strategy constant is used, this parameter tells the SimpleBaselineClassifier which value it should (always) predict. The value of the constant has to be present in the (target values of the) training data.

Required Method: __repr__

  • Functionality: Returns the formal string representation of the class. The method returns the following string:
"SimpleBaselineClassifier(strategy=<strategy>, random_state=<random_state>, constant=<constant>)"
  • Return:
    Type Description

Required Method: fit

  • Parameters:

    y_train numpy.ndarray (<num_train_samples>,) The target vector of the training set.

Required Method: predict

  • Functionality:

3. Implement the class KMeans (3P)

When performing machine learning tasks, the used data is not always labeled (no target variable). In such a case, a model that can make use of unlabeled data should be created. This is called unsupervised machine learning. One of the most common approaches is clustering, which is the grouping of data in such a way that similar data is assigned to the same group (cluster). K-means clustering is an algorithm to cluster data in a set of K distinct, non-overlapping clusters.
The class KMeans implements the k-means clustering algorithm. It uses Euclidean distance as the distance metric. Only use numpy and the standard library in your implementation of this class.
Note: There is a basic explanation here and a more thorough and mathematical one on Wikipedia.
Note: You can use sklearn.cluster.KMeans (with the same random state and n_clusters) to check whether your implementation works correctly (but do not use it in your implementation ;).

Required Method: __init__

  • Functionality:
    Store all parameters as non-public attributes.

  • Parameters:

    Name Type Default Value Description
    n_clusters int 8 Defines the number of clusters to form as well as the number of centroids.
    max_iter int 300 Maximum number of iterations of the k-means algorithm for a single run.
    tol float 1e-4 Relative tolerance with regards to Frobenius norm of the difference in the cluster centers of two consecutive iterations to declare convergence.
    random_state int None Determines random number generation for centroid initialization. If None no random state should be used.

Required Method: __repr__

  • Functionality: Returns the formal string representation of the class. The method returns the following string:
"KMeans(n_clusters=<n_clusters>, max_iter=<max_iter>, tol=<tolerance>, random_state=<random_state>)"
  • Return:
    Type Description
    str Formal string representation of the class

Required Method: fit

  • Functionality:
    This method fits the k-means algorithm by searching optimal cluster centers. This is done in the following way:

    1. Initialize cluster centers by taking n_cluster random points from the training samples.
    2. Calculate the Euclidean distance from each training sample to all cluster centers.
    3. Determine the closest cluster center of each training sample and assign each train sample to a cluster center.
    4. For each cluster center calculate the new mean center from all training samples assigned to the cluster center. Use this mean center as the new cluster center.
    5. Repeat step 2 to 4 until max_iter (maximum iterations) is reached or until the cumulative distance between step i and i+1 is smaller than tol (tolerance).

    Hint: You can use np.linalg.norm to calculate the Euclidean distance.
    It is adviced to save the final cluster centers in a non-public attribute since you need them in the predict method.
    Due to the small size of the used datasets, the classification process will finish rather quickly, independently of the efficiency of your implementation. It is however always advisable to think about which calculations can be performed using array operations instead of loops.

  • Parameters:

    Name Type Shape Description
    X_train numpy.ndarray (<num_train_samples>,<num_features>) The feature matrix of the training set.

Required Method: predict

  • Functionality:
    This method predicts the values of the target variable for the provided feature matrix of the test set (parameter X_test). This is done in the following steps:

    1. For each test sample, calculate the Euclidean distance to all cluster centers (fitted in the fit method).
    2. Assign the index of the cluster center with the smallest distance to the test sample as target variable.
      Hint: You can use np.linalg.norm to calculate the euclidean distance.

  • Parameters:

    Name Type Shape Description
    X_test numpy.ndarray (<num_test_samples>,<num_features>) The feature matrix of the test set.

  • Return:

    Type Shape Description
    numpy.ndarray (<num_test_samples>,) The predicted values of the target variable.
